# EduPilot Agent 프로젝트 요약

EduPilot Agent는 IT 교육 기관의 강사 및 교육 운영진의 업무 효율성을 향상시키고 수강생 관리를 자동화하기 위한 AI 기반 멀티 에이전트 시스템 포털입니다.

---

## 1. 프로젝트 개요
- **정의**: 강사의 강의 설계, 학생 관리, 평가 및 피드백, 학사 일정 및 공지 연동 업무를 보조하는 **AI 기반 교육 운영 보조 멀티 에이전트 플랫폼**입니다.
- **목표**:
  - 반복적인 행정 업무(커리큘럼 설계, 일정 동기화, 공지 작성, 상담 요약) 자동화로 강사의 리소스 절약.
  - 강의 교재 RAG 검색 및 코딩 과제/시험의 AI 자동 피드백을 통한 맞춤형 교육 환경 제공.
  - 3단 반응형 웹 대시보드(Streamlit + Tailwind CSS) 기반의 통합 워크페이스 구축.

---

## 2. 핵심 기능
- **Supervisor Agent (중앙 오케스트레이션)**: 사용자의 복합적인 자연어 질의를 분석하여 태스크를 분해하고 관련 에이전트들을 호출 및 조율.
- **Lecture Agent (RAG 강의 자료 검색)**: 로컬 PDF 교재(React, JS, Python, NCS 평가기준 등)를 벡터화(Chroma DB)하여 정확한 출처(파일명, 페이지 번호)와 함께 개념 답변 제공.
- **Student Agent (자연어-to-SQL 학생 관리)**: 학생 출결, 성적, 상담 내역, 진로 계획 등 SQLite DB 정보를 자연어 질문으로 조회할 수 있도록 SQL로 변환하여 처리.
- **Course Agent (교육과정 및 평가 설계)**: 주차별 핵심 커리큘럼 설계, 마크다운 가독성을 갖춘 강의 가이드라인 저작, 과목 평가용 5개 문항(객관식/주관식/코딩실습형) 자동 출제.
- **Grading Agent (AI 자동 평가 및 채점)**: 학생이 제출한 주관식 및 코딩 답안의 키워드 분석을 통해 부분 점수를 부여하고 맞춤형 AI 상세 피드백 리포트(JSON/XML 파싱) 제공.
- **Assignment Review Agent (코드 리뷰)**: 수강생이 제출한 소스코드의 구문 오류, 안티패턴을 감지하고 성능/가독성 개선안을 담은 상세 멘토링 리포트 작성.
- **Schedule & Notice Agents (일정 연동 및 공지 자동 생성)**: 학사 일정을 조회하고 이를 학생 및 학부모 전달용 LMS/메신저 공지글로 가공하는 템플릿 제작.
- **대한민국 법정 공휴일 자동 동기화**: 2026~2030년 공휴일을 내장하여 교육과정 시작일 및 강의 시간에 맞춰 실제 완료(종료)일을 실시간 자동 산출.

---

## 3. 기술 스택
- **Backend & Database**: Python 3.10+, SQLite3 (로컬 관계형 데이터베이스), `asyncio`
- **AI / LLM & RAG**: LangChain, langchain-openai, OpenAI API (`gpt-4o-mini`, `text-embedding-3-small`), Chroma DB (로컬 벡터 데이터베이스), `pypdf`
- **Frontend / UI**: Streamlit (3단 레이아웃), Tailwind CSS v3 (CDN 연동 및 Custom CSS 주입)
- **Environment**: `python-dotenv`, `pandas`, `pydantic`

---

## 4. 적용 기술
- **Hub-and-Spoke 멀티 에이전트 구조**: 중앙 허브(Supervisor)가 실행 계획을 수립하고 각 영역의 전문가 에이전트(Worker)에게 태스크를 라우팅하는 분산형 아키텍처.
- **비동기 태스크 병렬 처리 (Async Task Parallelism)**: 독립적인 에이전트 호출을 `asyncio.gather`를 통해 동시 수행하여 LLM 응답 대기 시간(Latency)을 40% 이상 개선.
- **Text-to-SQL & Dynamic Querying**: 자연어 의도를 SQLite 스키마에 적합한 쿼리로 변환하여 정형 데이터에 안전하고 정확하게 직접 접근.
- **정밀 RAG & Source Tracking**: 텍스트 청킹(RecursiveCharacterTextSplitter) 기호 및 메타데이터를 역추적하여 답변 하단에 정확한 출처(예: `[출처: Python.pdf (23p)]`)를 명시, 환각(Hallucination) 방지.
- **구조화된 출력 제어 (Structured Output Control)**: LLM이 XML/JSON 규격을 엄격히 따르도록 프롬프트를 설계하고, 이를 파싱하여 Streamlit 상에서 미려한 카드와 스코어카드로 렌더링.

---

## 5. 핵심 성과
- **행정 리소스 대폭 경감**: 커리큘럼 설계, 일정 조율, 공강 처리, 강의 가이드 저작 등 기존에 장시간 소요되던 강사의 문서 작업을 단일 플랫폼 내 자동화.
- **피드백 사이클 단축**: 온라인 시험 응시 즉시 AI가 답안 채점, 정오 판단, 상세 해설, 리팩토링 제안을 포함한 성적 리포트를 즉시 생성하여 학습 피드백 가속화.
- **하이브리드 정보 처리 구현**: RAG(교재 PDF)와 RDBMS(학생/일정 데이터)의 동시 조회가 필요한 복합적인 질문을 단 하나의 챗봇 대화창에서 원활하게 완수.
- **Tailwind CSS를 활용한 고품질 웹 대시보드**: Streamlit의 투박한 레이아웃을 극복하고, 고정형 Navbar, 역할(교사/학생)에 따른 화면 분리, 모니터링 타임라인 등을 미려하게 구현하여 높은 사용자 경험(UX) 확보.

---

## 6. 한계점 및 향후 계획
- **인증 및 보안 한계**: 현재는 단순 평문 아이디 검증과 고정 패스워드('1111')를 사용하고 있어, JWT 토큰 및 암호화 해싱을 사용한 세션 관리 시스템 도입 필요.
- **대량의 PDF 업로드 성능**: 교재 임베딩 처리가 동기로 실행될 때 브라우저 멈춤 방지를 위해 Celery/Redis 등의 비동기 태스크 큐 도입 필요.
- **대화 컨텍스트(Memory) 유지 개선**: 장기적인 대화 기억 장치 및 이전 단계의 태스크 피드백 루프를 강화하는 에이전트 메모리 컴포넌트 추가 예정.
- **외부 채널 실 연동**: 템플릿 생성 수준을 넘어 실제 카카오톡 알림톡/LMS API 연동 및 GitHub Webhook 연동을 통한 자동 과제 수집·코드 리뷰 기능 확장.
